1. Introduction

The Mobile Calculator Application is a robust and user-friendly Android app designed to handle basic arithmetic operations, including addition, subtraction, multiplication, and division. The app is built using the MVVM (Model-View-ViewModel) architectural pattern to ensure a clean, modular, and maintainable codebase. The primary goal of this project is to provide users with a reliable tool for quick calculations while demonstrating good software development practices.

2. Application of Joel Marsh's UX Principles

The Calculator application relies on UX principles described by Joel Marsh to provide an optimal user experience. These principles focus on simplicity, clarity, user feedback, visual hierarchy, and accessibility.

1. Simplicity

The calculator interface is minimalist, with only the buttons necessary to perform basic calculations. This allows the user to focus solely on their tasks.

Portrait mode

Landscape Mode

2. Clarity

Each button is clearly labeled, and a visual hierarchy is used to make primary operations more visible.

3. User Feedback

When an operation is performed, a visual effect signals to the user that their action has been acknowledged. In case of an error, a clear message is displayed, such as "Error: Division by zero."

4. Hierarchy and Priority

Key elements, such as the result screen, are placed at the top, while the calculator buttons are positioned at the bottom for intuitive use.

5. Accessibility

The application uses high-contrast colors and large buttons to ensure easy use, even for users with physical limitations.

3. Methodology

Why MVVM

The Model-View-ViewModel (MVVM) architecture is a modern design pattern that separates the business logic (Model) from the user interface (View) through a ViewModel, which acts as a mediator. This pattern is especially useful in Android development, where maintaining a separation of concerns can enhance testability, reusability, and maintainability.

5. Critical Analysis

While the Calculator application provides a solid and functional base, it presents some limitations that could be improved to offer a better user experience. Here are the main points of critique:

1. Lack of Calculation History

The application does not retain a history of performed calculations. This functionality is crucial for users who want to review previous calculations or make corrections. Adding a visible and interactive history would significantly enhance usability.

2. Lack of Real-Time Expression Preview

The user cannot see the mathematical expression they are entering, which makes it difficult to verify errors before performing the calculation. A dedicated text field or real-time display of the complete expression would improve clarity and precision.

3. Functional Limitations

Currently, the application only handles basic arithmetic operations (addition, subtraction, multiplication, division). It does not support advanced features such as:

  • Scientific calculations (trigonometry, logarithms, powers, etc.)
  • Support for parentheses or operation precedence
  • Handling large numbers or complex numbers

6. Internationalisation

The Calculator app is easy for everyone to use because it works with numbers. Numbers are the same in all languages, so there is no need to translate anything.

The app only uses basic math operations like addition, subtraction, multiplication, and division. This makes it simple for people from any country to use without needing words in their language.

This makes the calculator useful for everyone around the world, making it easy and inclusive.

7. Conclusion

The Calculator app is simple and useful for basic math. It is easy to use and works for everyone because numbers are the same in all languages.

The app has some problems, like no history of calculations and no way to see the full math expression while typing. It also does not have advanced features like scientific calculations.

This project was a great way to learn many things. I learned about MVVM architecture, how to create a good user experience (UX), and how to use Android Studio to make an app. These skills will help me in future projects.

This project shows a good understanding of how to make apps, but it can be improved in the future by adding more features and making it better for users.
